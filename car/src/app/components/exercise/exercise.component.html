Exercise page

    Let’s start the party! 🥳

    • 1.) Use CarService to communicate with API and get first preconfigured cars data. We have already prepared a test API call GET /car, you can see tha data in console after visiting of Exercise page.

    • 2.) Find app/models/car.vo.ts file and prepare your Car model depends on parameters, that you have got from http://localhost:3000/car response:

      • id: number
      • name: string
      • …
      • …

    • 3.) Create a new page, where we will display all our cars and bind this page to “Cars” navigation link. This page should also include search field, where you can filter list of displayed cars (against api).

      Filter endpoint GET /car?q=searchTerm , where searchTerm is the filter value of the search input

    • 4.) Create card component preview – this component will be used to show short information of a single car, in order to use it later in our “Cars” page. This card component should be be also able to display dynamic content, provided by parent component, to increase component reusability.

      Content that must be shown in card:

      • Image - car image (Chose a random image from assets)
      • Model - model of car
      • FirstRegistration - registration date
      • Some other car details
      • Delete button – used to remove our car
      • Details button – navigate to car details page, where will be displayed all car information

    • 5.) Display preconfigured cars with help of PreviewCardComponent on our “Cars” page.

    • 6.) In assets folder you can find some cars images that you can randomly use and insert in your cards.

    • 7.) Create new Component, that will be used to create new cars, based on car model (interface), and bind it to new navigation link (“Add car”)

      Create car form:

      • Model field is required
      • FirstRegistration field is required and should be in "DD/MM/YYYY" format
      • Origin field is required and must be of type "GERMANY", "USA" or other origin.
      • Save button must be active if all fields are valid

      Please provide error messages for a better user experience.

    • 8.) Consume POST endpoint to create new car.

    • 9.) Create CarDetailsComponent to show car information. For navigation must be used our “Details” button from car card.

    • 10.) Consume DELETE endpoint to remove car from “Cars” page at click on “Delete” button of car card.
